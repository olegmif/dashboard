## Журнал
- **Ph1 — Настройки конкретного виджета: готово.** Механизм per-widget опций:
  `config.Widget.Options toml.Primitive` (сырая подтаблица `[widget.options]`) +
  приватная копия `toml.MetaData` (стат в `Load`) + метод `DecodeOptions(target)`
  для ленивого декода в типизированный struct виджета. `main` про содержимое
  опций не знает. Отсутствие опций → no-op (дефолты сохраняются; проверено
  поведение `unifyStruct(nil)` в BurntSushi). Юнит-тест `config_test.go`
  покрывает обе ветки (с опциями / без). `config.example.toml` документирует
  форму. Поведение приложения не изменилось — чистый «водопровод» под Ph2/Ph3.

## Сделано
- `internal/config` — поле `Options`, `DecodeOptions`, стат метаданных в `Load`.
- `internal/config/config_test.go` — тест декодирования опций.
- `config.example.toml` — документация формы `[widget.options]`.

## Решения, принятые по ходу
- **Механизм per-widget настроек: РЕШЕНО** — `[widget.options]` →
  `toml.Primitive` → `DecodeOptions(&struct)`. Каждый виджет декодирует свои
  опции сам в `New`, дефолты задаёт до декода.
- Фоторамка: режим вписывания (cover/contain), порядок (seq/random),
  рекурсивность скана: _не решено_.
