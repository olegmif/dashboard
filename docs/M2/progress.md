**Статус:** ✅ M2 завершён — Ph1–Ph5 готовы (часы + фоторамка)

## Журнал
- **Ph1 — Настройки конкретного виджета: готово.** Механизм per-widget опций:
  `config.Widget.Options toml.Primitive` (сырая подтаблица `[widget.options]`) +
  приватная копия `toml.MetaData` (стат в `Load`) + метод `DecodeOptions(target)`
  для ленивого декода в типизированный struct виджета. `main` про содержимое
  опций не знает. Отсутствие опций → no-op (дефолты сохраняются; проверено
  поведение `unifyStruct(nil)` в BurntSushi). Юнит-тест `config_test.go`
  покрывает обе ветки (с опциями / без). `config.example.toml` документирует
  форму. Поведение приложения не изменилось — чистый «водопровод» под Ph2/Ph3.

- **Ph2 — Часы: готово (проверено пользователем).** Полноценный виджет на
  каркасе:
  - Опции `format` / `date_format` (Go reference-layout) через механизм Ph1, с
    дефолтами и `Warn` при кривых опциях. Пустой `date_format` убирает строку даты.
  - Двухстрочный layout (время + дата), вертикальный `Box`, центрирование контента.
  - Механика тика (горутина → канал → `OnMain`) — без изменений, эталон Ph4.
  - **Вид доведён по ходу (правки сверх плана):**
    - **прозрачный фон у ВСЕХ окон** (`window { background: transparent }`) —
      решено убрать серую подложку по умолчанию; виджет, которому нужен фон, сам
      рисует прямоугольник-подложку на окно. picom НЕ трогаем — он лишь запущен,
      прозрачность это per-pixel alpha от GTK.
    - часы: белый текст, крупный (время 96px, дата 30px), без карточки.
    - окно часов вручную отцентрировано на левом мониторе (1920×1080@+0+0):
      `620×240` в `(650,420)`. Авто-центрирование (опцию) пользователь отклонил —
      координаты в `config.toml`.

- **Ph3–Ph5 — Фоторамка: готово (проверено пользователем).** Сделаны вместе
  (ротатор без показа не посмотреть):
  - Опции `dirs` (список), `recursive`, `interval` (сек), `order`
    (`sequential`/`random`).
  - Скан каталогов с фильтром по расширениям (jpg/jpeg/png/webp/gif/bmp),
    сортировка; битые/отсутствующие каталоги → `Warn` + пропуск, не падаем.
    Нет фото → статус-текст.
  - Ротатор: горутина-продюсер выбирает следующий файл по таймеру.
  - **Декод+ресайз под размер окна — в фоне** (`NewPixbufFromFileAtScale`),
    готовый pixbuf → `OnMain` → `Gtk.Picture` (эталон дисциплины с тяжёлой
    работой в фоне).
  - Режим вписывания: пробовали `Contain`, остановились на **`Cover`**.
  - По ходу: остальные виджеты (weather/sysicons/telegram) скрыты
    (закомментированы в `config.toml`); рамка вертикальная **2:3** `450×675`.

- **Матовое стекло (frosted glass) — готово, очень понравилось.** Стиль доведён
  по ходу сверх плана:
  - **Тут впервые тронули picom-конфиг** (ранее обещали не трогать — для простой
    прозрачности и не требовалось, а для размытия требуется): включили
    `blur-method = dual_kawase` + `blur-background` точечно по `class_g='dashboard'`.
  - CSS-класс `.glass` (любой виджет может взять): полупрозрачный тон
    `rgba(140,140,156,0.42)` + белый веер (`linear-gradient`) для имитации
    macOS-подсветки + светлая верхняя кромка и тень снизу (`box-shadow inset`).
    Скругление углов — от picom (по классу), CSS не дублирует.
  - **Часы**: стеклянная панель на всё окно, белый текст по центру.
  - **Фоторамка**: фото с паддингом `12px` → стеклянное **паспарту** вокруг
    снимка (за непрозрачным фото стекла не видно — поэтому именно поля).

## Решения, принятые по ходу
- **Механизм per-widget настроек: РЕШЕНО** — `[widget.options]` →
  `toml.Primitive` → `DecodeOptions(&struct)`. Каждый виджет декодирует свои
  опции сам в `New`, дефолты задаёт до декода.
- **Фон окон: РЕШЕНО — прозрачный по умолчанию у всех.** Серой подложки у
  `.widget` больше нет. Кому нужен фон — берёт `.glass` (или свою подложку).
- **Матовое стекло: РЕШЕНО — `.glass` + picom blur по классу.** Для размытия
  picom-конфиг всё же тронули (включили blur точечно для `class_g='dashboard'`).
  Это обновляет прежнее «picom не трогаем» — оно держалось для простой
  прозрачности, но не для blur. Чужие окна не затронуты.
- **Центрирование окна: РЕШЕНО — вручную через координаты** в `config.toml`
  (опцию авто-центрирования отклонили). Авто-детект монитора по-прежнему
  отложен (возможный M1.Ph2.1).
- **Фоторамка: РЕШЕНО** — режим **`Cover`**, порядок по умолчанию `sequential`
  (есть `random`), рекурсивность по умолчанию ВЫКЛ. Аспект окна 2:3.
