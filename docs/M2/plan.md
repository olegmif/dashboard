# M2 — Часы + фоторамка

**Цель:** два первых «живых» виджета на каркасе M1. Часы уже частично тикают
(носитель Ph4-паттерна из M1) — доводим до полноценного виджета; фоторамку делаем
с нуля. Параллельно закрываем первый общий пробел — **настройки конкретного
виджета** (каталоги фото, формат часов).

## Фазы

### Ph1 — Настройки конкретного виджета (общий шов)
`config.Widget` сейчас знает только `type/instance/геометрию`. Фоторамке нужны
каталоги + интервал, часам — формат. Нужен generic способ передать виджету его
опции, не засоряя общий конфиг и не делая `main` осведомлённым о деталях.

- В `[[widget]]` — подтаблица `[widget.options]` с произвольными ключами.
- `config.Widget` получает поле `Options` (сырое — `toml.Primitive`), которое
  каждый виджет декодирует в свой типизированный struct.
- Хелпер декодирования `options → struct` (в `config` или `widget`); виджет
  зовёт его в своём `New`. `main` содержимого options не знает — расширяемость
  реестра сохраняется.
- Отсутствие/битые options → дефолты + `Warn`, не падать.

### Ph2 — Часы (довести до виджета)
Механику тика (горутина → канал → `OnMain`) НЕ трогаем — она уже эталон. Доводим
содержание и вид:
- Формат времени и флаг «показывать дату/день недели» — из options (с дефолтами).
- Второй ряд: дата / день недели.
- CSS: моноширинный шрифт для цифр, размер, класс `widget-clock`.

### Ph3 — Фоторамка: источник
- Каталог(и) из options (`dirs = [...]`).
- Скан файлов с фильтром по расширениям (jpg/jpeg/png/webp/…). Рекурсивность —
  опция (по умолчанию НЕ рекурсивно).
- Пустой/несуществующий каталог → `Warn`, виджет показывает «нет фото», не падает.

### Ph4 — Фоторамка: ротатор
- Горутина-продюсер: раз в N сек (из options) выбирает следующий файл —
  последовательно или случайно (опция `order`).
- Путь едет в канал → консьюмер применяет через `OnMain` (эталон Ph4 из M1).

### Ph5 — Фоторамка: загрузка и ресайз
- **Тяжёлый декод/ресайз — в горутине-продюсере, НЕ на главном потоке.**
  `GdkPixbuf.NewFromFileAtScale(path, w, h, preserveAspect)` под размер окна
  (фото бывают 4000×3000 — иначе спайки CPU/RAM на декоде).
- Готовый pixbuf передаём на главный поток (`OnMain`) и ставим в `Gtk.Picture`
  (или `Gtk.Image`). Это уточняет эталон: продюсер делает тяжёлую работу, UI
  получает готовое.
- Режим вписывания (cover/contain) — выбрать при реализации.

## Критерий готовности
Часы показывают время и дату нужным шрифтом; фоторамка циклически меняет фото из
каталога(ов) по таймеру без заметных спайков CPU/RAM (ресайз до показа в фоне).

## Зависимости
- Ph1 разблокирует Ph2 (формат) и Ph3–Ph5 (каталоги/интервал/порядок).

## Вне объёма (перенесено)
- Плавный кросс-фейд при смене фото → M7.
