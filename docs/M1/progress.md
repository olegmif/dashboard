## Журнал
- **Ph1 — Пустое окно: готово.** Окружение: Go 1.26, GTK4 4.22.4, GCC 16.1,
  pkg-config 2.5. Создан Go-модуль `github.com/olegmif/dashboard`, `main.go` с
  минимальным GTK4-окном через `diamondburned/gotk4` (400×300, `SetDecorated(false)`).
  Сборка `go build -o dashboard .` проходит; при сборке cgo сыпет безобидные
  warning'и `conflicting types for 'free'` (особенность gotk4 на GCC 16, не ошибки).
  Бинарь запускается — окно отображается. Стек GTK4+gotk4+cgo подтверждён рабочим.
  - Замечание: `go build` всё же качал зависимости по сети, хотя ожидалось, что
    `go mod tidy` уже наполнил кэш. Разовое наполнение кэша; проверить офлайн-сборкой
    `GOPROXY=off go build`.
  - При запуске GTK4 пишет WARN про `xdg-desktop-portal` (`org.freedesktop.portal.Desktop`
    не активен) — портал в dwm не запущен и виджету не нужен. Безвредно; глушится
    `GTK_USE_PORTAL=0`. Чистку лог-шума запланировать на Ph3 (каркас).

- **Ph2 — Размещение по координатам: готово.** Реализован override-redirect через
  cgo/Xlib, спрятанный за бэкенд-интерфейсом `internal/deskwindow` (по просьбе:
  `Backend`/`Geometry` портативны, X11-специфика в `x11.go` с `//go:build linux`,
  `New()` — точка будущего выбора бэкенда). Тайминг: флаг override-redirect
  ставится на `realize` (до маппинга), позиция/размер/`XLowerWindow` — на `map`.
  Координаты из `config.toml` (`[window]`), парсер `BurntSushi/toml`.
  **Проверено пользователем на реальной системе (3 монитора, dwm):** окно
  появляется на левом мониторе в заданном прямоугольнике, dwm его не тайлит,
  фокус не ворует. ✅ Главный риск M1 снят.
  - Доп. план-правка по ходу: автоопределение монитора решено НЕ делать (GTK4 сам
    не позиционирует, dwm для managed-окон диктует геометрию) → override-redirect +
    координаты из конфига. Авто-детект помечен как возможный Ph2.1 на будущее.
  - **Скруглённые углы через picom: работают (проверено пользователем).** Окну
    проставлен `WM_CLASS = dashboard` (`XSetClassHint` в `x11.go`); в
    `~/.config/picom/picom.conf` добавлено правило `rules` (picom 13):
    `match = "class_g = 'dashboard'"; corner-radius = 12;`. Глобальный
    `corner-radius = 0` оставлен — скругляется только дашборд. Полупрозрачность
    тела пока НЕ включаем (по решению: только скругление). picom композитит и
    скругляет override-redirect окно нормально.

- **Ph3 — Мультиоконный каркас: готово (проверено пользователем — пять окон).**
  Модель «виджет = окно» реализована:
  - `config.toml` → массив `[[widget]]` (type/instance/геометрия); `internal/config`
    читает список, есть fallback на `Default()` если пусто/нет файла.
  - `internal/widget` — контракт `Builder = func(config.Widget)(gtk.Widgetter,error)`,
    `Registry` (имя→конструктор), хелпер `Placeholder`. Реестр собирается явно в
    `main` (без init-магии/blank-import).
  - Пять подпакетов-заглушек: `widget/{clock,photo,telegram,weather,sysicons}`.
  - Главный цикл в `main`: config → окно (`SetDecorated(false)`, размер) → виджет
    из реестра → `backend.Pin(win, geo, instance)` → `Present`. Невалидные/
    неизвестные записи пропускаются с `Warn`.
  - `instance` прокидывается в WM_CLASS-instance (класс общий `dashboard`),
    плюс в заголовок `dashboard:<instance>`.
  - `style.css` встроен через `go:embed`, ставится на дисплей одним
    `GtkCssProvider` (приоритет APPLICATION).
  - Прозрачность НЕ включена (по решению) — окна непрозрачны, скругление от picom.
  - Проверено: пять окон встают по координатам, не тайлятся, фокус не воруют;
    добавление/перенос виджета — правкой `config.toml` без пересборки.

- **Ph4 — Дисциплина обновлений: готово (проверено — часы тикают).** Эталон
  «фон → UI» реализован:
  - `widget.OnMain(func())` — обёртка над `glib.IdleAdd`, единственный шов, через
    который фоновая горутина трогает виджет (GTK-виджеты меняем только на главном
    потоке).
  - Полная форма паттерна — в `widget/clock`: продюсер-горутина шлёт время в
    канал, консьюмер-горутина применяет каждое значение через `OnMain`. Помечено
    комментарием «copy this shape» для будущих виджетов.
  - Часы тикают `ЧЧ:ММ:СС` раз в секунду — живое подтверждение, что дисциплина
    работает и UI не падает.
  - Честная пометка: живые часы формально относятся к M2; здесь они в минимальном
    виде как носитель Ph4-паттерна. Шрифты/дата/формат — в M2 (clock-заглушка уже
    заменена рабочим тиком, M2 её допилит, а не создаёт заново).

## Сделано
- `go.mod` / `go.sum` / `.gitignore` / `main.go` — каркас сборки + главный цикл.
- `internal/deskwindow` — бэкенд-интерфейс + X11 override-redirect (cgo).
- `internal/widget` (+5 подпакетов) — контракт, реестр, заглушки.
- `style.css`, `config.toml`/`config.example.toml` — стиль и список виджетов.
  Окну детерминированно проставляется `WM_CLASS = dashboard` (через `XSetClassHint`),
  чтобы picom/WM-правила цеплялись по `class_g`, а не по непредсказуемому
  автоклассу GTK. Скругление углов — правилом picom по `class_g = 'dashboard'`
  (тело окна непрозрачное; полупрозрачность пока не включаем).
- `internal/config` + `config.toml`/`config.example.toml` — геометрия окна.
- Подтверждено: GTK4 рисуется через gotk4 И прибивается к монитору под dwm.

## Решения, принятые по ходу
- **GTK4 vs GTK3: GTK4 РАБОТАЕТ — остаёмся на GTK4.** Окно рисуется и успешно
  прибивается к левому монитору под dwm через override-redirect. Откат на GTK3
  не требуется. (Открытый вопрос закрыт.)
- **Способ «прибить» окно: override-redirect** (вариант A). Самый портативный —
  работает на любом X11-WM, не только dwm. Спрятан за бэкенд-интерфейсом.
- **Модель окон: один виджет = одно окно** (не единая панель). N независимых
  override-redirect окон, геометрия каждого — в `config.toml`. Перенос виджета
  (в т.ч. на другой монитор) = смена координат. Каждое окно может быть прозрачным
  при запущенном композиторе (picom) → скруглённые углы, полупрозрачный фон;
  без композитора деградируем к непрозрачному. Реализация мультиоконности — Ph3.
- **Фокус (характеристика override-redirect, проверять не требуется):** мышь
  (клики/скролл/hover) до окна **доходит** (хватит для read-only Telegram с
  прокруткой); клавиатурный фокус dwm не отдаёт — нашим виджетам он и не нужен.
