## Журнал
- **Ph1 — Пустое окно: готово.** Окружение: Go 1.26, GTK4 4.22.4, GCC 16.1,
  pkg-config 2.5. Создан Go-модуль `github.com/olegmif/dashboard`, `main.go` с
  минимальным GTK4-окном через `diamondburned/gotk4` (400×300, `SetDecorated(false)`).
  Сборка `go build -o dashboard .` проходит; при сборке cgo сыпет безобидные
  warning'и `conflicting types for 'free'` (особенность gotk4 на GCC 16, не ошибки).
  Бинарь запускается — окно отображается. Стек GTK4+gotk4+cgo подтверждён рабочим.
  - Замечание: `go build` всё же качал зависимости по сети, хотя ожидалось, что
    `go mod tidy` уже наполнил кэш. Разовое наполнение кэша; проверить офлайн-сборкой
    `GOPROXY=off go build`.
  - При запуске GTK4 пишет WARN про `xdg-desktop-portal` (`org.freedesktop.portal.Desktop`
    не активен) — портал в dwm не запущен и виджету не нужен. Безвредно; глушится
    `GTK_USE_PORTAL=0`. Чистку лог-шума запланировать на Ph3 (каркас).

- **Ph2 — Размещение по координатам: готово.** Реализован override-redirect через
  cgo/Xlib, спрятанный за бэкенд-интерфейсом `internal/deskwindow` (по просьбе:
  `Backend`/`Geometry` портативны, X11-специфика в `x11.go` с `//go:build linux`,
  `New()` — точка будущего выбора бэкенда). Тайминг: флаг override-redirect
  ставится на `realize` (до маппинга), позиция/размер/`XLowerWindow` — на `map`.
  Координаты из `config.toml` (`[window]`), парсер `BurntSushi/toml`.
  **Проверено пользователем на реальной системе (3 монитора, dwm):** окно
  появляется на левом мониторе в заданном прямоугольнике, dwm его не тайлит,
  фокус не ворует. ✅ Главный риск M1 снят.
  - Доп. план-правка по ходу: автоопределение монитора решено НЕ делать (GTK4 сам
    не позиционирует, dwm для managed-окон диктует геометрию) → override-redirect +
    координаты из конфига. Авто-детект помечен как возможный Ph2.1 на будущее.

## Сделано
- `go.mod` / `go.sum` / `.gitignore` / `main.go` — каркас сборки.
- `internal/deskwindow` — бэкенд-интерфейс + X11 override-redirect (cgo).
- `internal/config` + `config.toml`/`config.example.toml` — геометрия окна.
- Подтверждено: GTK4 рисуется через gotk4 И прибивается к монитору под dwm.

## Решения, принятые по ходу
- **GTK4 vs GTK3: GTK4 РАБОТАЕТ — остаёмся на GTK4.** Окно рисуется и успешно
  прибивается к левому монитору под dwm через override-redirect. Откат на GTK3
  не требуется. (Открытый вопрос закрыт.)
- **Способ «прибить» окно: override-redirect** (вариант A). Самый портативный —
  работает на любом X11-WM, не только dwm. Спрятан за бэкенд-интерфейсом.
- Одно окно или несколько окон-виджетов: _не решено_ (решим в Ph3, каркас layout).
